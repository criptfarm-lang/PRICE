# Fish to Business

Единый Node.js сервер для управления прайс-листом и мониторинга цен через МойСклад.

## Структура

```
fish-to-business/
├── server.js          # Единый сервер (все маршруты)
├── package.json
├── README.md
└── public/
    ├── monitor.html       # Внутренний монитор цен (тёмная тема)
    ├── admin.html         # Управление прайс-листом
    └── price-monitor.html # Новый монитор (работает напрямую через браузер)
```

## Страницы

| URL | Описание |
|-----|----------|
| `/` | Монитор цен — таблица товаров, остатки, наценки, себестоимость |
| `/admin` | Панель управления прайс-листом Fish to Business |
| `/price-monitor` | Новый монитор ассортимента (токен вводится в браузере) |

## API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/data` | Все товары с ценами, остатками и продажами из МС |
| PUT | `/api/price` | Обновить цену товара в МойСклад |
| PUT | `/api/archive` | Архивировать товар в МойСклад |
| GET/POST | `/api/settings` | Настройки монитора (порядок строк, цены конкурентов) |
| GET/POST | `/api/pricelist` | Данные прайс-листа (категории, товары, настройки компании) |
| GET | `/api/moysklad/product?code=XXX` | Найти товар в МС по коду |
| POST | `/api/moysklad/sync` | Обновить цены в прайсе из МС |
| POST | `/api/moysklad/test` | Проверить подключение (login/password) |

## Запуск

### Локально

```bash
MOYSKLAD_TOKEN=ваш_токен node server.js
```

### Railway / Render

Добавьте переменную окружения:
```
MOYSKLAD_TOKEN=ваш_bearer_токен_моysклад
```

Опционально:
```
PORT=3000
DATA_DIR=/app/data
```

## Данные

Сервер хранит два JSON-файла в `DATA_DIR` (по умолчанию `/app/data`):

- `pricelist.json` — данные прайс-листа (категории, товары, настройки компании)
- `monitor-data.json` — настройки монитора (порядок строк, цены конкурентов)
